# Activity Completion Checklist

Use this checklist to ensure you have sufficient evidence for each activity.

## For Each Activity, Verify:

### Documentation (Must Have)
- [ ] README.md file explaining the activity
- [ ] Clear description of activity objectives
- [ ] Step-by-step explanation of how you completed it
- [ ] Tools and technologies used listed
- [ ] Key learnings documented

### Evidence (Must Have)
- [ ] Source code or scripts in `/code` folder
- [ ] Screenshots of execution in `/screenshots` folder
- [ ] Command outputs or logs showing successful completion
- [ ] Any configuration files used
- [ ] Results or findings documented

### Supporting Materials (Recommended)
- [ ] Links to resources used
- [ ] Challenges faced and how you resolved them
- [ ] Before/after comparisons where applicable
- [ ] Any certificates or support letters

### Completeness Check
- [ ] Sufficient detail to prove activity was completed
- [ ] Evidence clearly shows your work, not just copied
- [ ] Documentation explains how you did it, not just what was done
- [ ] All code/scripts are commented and explained
- [ ] Screenshots are clear and labeled

## Before Submission

- [ ] All activities have README files
- [ ] All activities have supporting evidence in code/ and screenshots/
- [ ] GitHub commit history shows regular progress
- [ ] Progress tracker is up to date
- [ ] No sensitive information in repository (passwords, API keys, etc.)
- [ ] Repository is ready for PDF report generation
